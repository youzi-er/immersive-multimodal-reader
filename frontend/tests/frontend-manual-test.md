# 前端手动测试

本文档用于记录沉浸式多模态阅读器前端页面的手动测试内容。前端手动测试主要验证用户在浏览器中的实际操作流程是否正常，包括登录注册、章节阅读、线索收集和聊天助手四个核心流程。

## 1. 登录注册流程测试

### 测试目标

验证用户可以正常登录、注册，并且在账号或密码错误时，页面能够显示错误提示。

### 测试步骤

1. 打开系统首页。
2. 点击页面上的“登录”入口。
3. 输入正确账号和密码。
   - 用户名：`demo`
   - 密码：`123456`
4. 点击登录按钮。
5. 检查页面是否进入书架页面。
6. 点击退出登录。
7. 再次进入登录页面。
8. 输入正确用户名，但输入错误密码。
9. 点击登录按钮。
10. 检查页面是否显示登录失败提示。
11. 进入注册页面。
12. 输入新用户信息。
    - 昵称：自定义
    - 用户名：不少于 3 个字符
    - 密码：不少于 6 位
13. 点击注册按钮。
14. 检查注册成功后是否进入书架页面。

### 预期结果

1. 使用正确账号和密码可以登录成功。
2. 登录成功后页面进入书架页面。
3. 使用错误密码登录时，页面显示错误提示。
4. 注册信息符合要求时，可以注册成功。
5. 注册成功后页面进入书架页面。
6. 整个过程中页面没有白屏、卡死或明显报错。

### 实际结果

页面正常进入书架，错误密码时显示提示。

### 测试结论

通过。

## 2. 章节阅读流程测试

### 测试目标

验证用户登录后可以进入阅读器，并且可以正常查看章节标题、正文内容和切换章节。

### 测试步骤

1. 登录系统。
2. 进入书架页面。
3. 点击“继续阅读”或当前书籍入口。
4. 检查是否进入阅读页面。
5. 检查阅读页面是否显示当前章节标题。
6. 检查阅读页面是否显示正文内容。
7. 点击左侧章节列表中的其他章节。
8. 检查章节标题和正文内容是否发生切换。
9. 点击“上一章”按钮。
10. 检查是否切换到上一章。
11. 点击“下一章”按钮。
12. 检查是否切换到下一章。

### 预期结果

1. 用户可以从书架进入阅读页面。
2. 阅读页面能够正常显示章节标题。
3. 阅读页面能够正常显示正文内容。
4. 点击章节列表后，正文内容会切换到对应章节。
5. “上一章”和“下一章”按钮可以正常切换章节。
6. 当前已经是第一章或最后一章时，对应按钮应不可用或不会造成页面错误。

### 实际结果

页面显示正常，页面切换正常。

### 测试结论

通过。

## 3. 线索收集流程测试

### 测试目标

验证用户在阅读过程中可以收集线索，并且已收集的线索可以在证物袋中查看和保存。

### 测试步骤

1. 登录系统。
2. 进入阅读页面。
3. 在正文中寻找可以点击的线索文字。
4. 点击线索文字。
5. 检查页面是否出现线索收集提示。
6. 检查证物袋数量是否增加。
7. 打开证物袋。
8. 检查证物袋中是否显示刚刚收集的线索。
9. 检查线索是否包含名称、类型和描述信息。
10. 刷新页面。
11. 再次打开证物袋。
12. 检查已收集线索是否仍然存在。
13. 再次点击同一个线索。
14. 检查系统是否避免重复添加同一条线索。

### 预期结果

1. 点击正文中的线索后，系统可以成功收集线索。
2. 页面能够显示线索收集提示。
3. 证物袋数量会更新。
4. 证物袋中可以看到已收集线索。
5. 线索信息包含名称、类型和描述。
6. 刷新页面后，已收集线索不会丢失。
7. 重复点击同一条线索时，不会重复添加。

### 实际结果

可正常收集线索，并可在证物袋中查看和保存。

### 测试结论

通过。

## 4. 聊天助手流程测试

### 测试目标

验证阅读页面中的聊天助手可以接收用户问题，并显示助手回答。

### 测试步骤

1. 登录系统。
2. 进入阅读页面。
3. 点击右侧辅助区域中的“助手”选项。
4. 在输入框中输入一个问题。
   - 示例：`目前有哪些线索？`
5. 点击发送按钮，或按回车发送。
6. 检查聊天区域是否显示用户刚刚输入的问题。
7. 等待助手回答。
8. 检查聊天区域是否显示助手回复。
9. 清空输入框。
10. 尝试发送空内容。
11. 检查系统是否阻止发送空问题。

### 预期结果

1. 用户可以正常输入并发送问题。
2. 页面会显示用户发送的问题。
3. 后端返回后，页面会显示助手回答。
4. 空问题不会被发送。
5. 如果 AI 接口失败，系统不会白屏或崩溃。
6. 在 AI 接口失败时，系统应使用普通聊天接口返回结果，或显示合理提示。

### 实际结果

未接大模型：聊天助手基础问答功能可用；当前环境下真实大模型接口未成功调用，系统自动回退到普通聊天接口，页面未崩溃。
已接大模型：聊天助手基础问答流程正常，真实大模型接口已成功接入。

### 测试结论

通过。

## 5. 语音、图片、多模态功能测试

### 测试目标

验证阅读页面中的语音朗读、场景图片生成和多模态辅助功能是否具备可测试入口，并记录当前阶段是否接入真实 AI 服务。

### 测试状态

已接入API KEY

### 测试步骤

1. 登录系统。
2. 进入阅读页面。
3. 点击正文中的人物对话内容。
4. 检查是否出现语音播放按钮。
5. 点击播放按钮。
6. 检查是否可以播放语音，或在接口失败时使用浏览器语音兜底。
7. 切换到右侧“现场”区域。
8. 点击“生成场景图”按钮。
9. 检查是否显示 AI 生成图片，或在生成失败时显示本地示意图。
10. 检查页面是否显示合理提示，例如接口失败、使用兜底方案等。

### 预期结果

1. 点击对话后可以看到语音播放入口。
2. 语音接口正常时，可以播放生成的语音。
3. 语音接口失败时，页面不会崩溃，可以使用浏览器语音兜底或显示合理提示。
4. 点击生成场景图后，系统可以请求图片生成接口。
5. 图片接口正常时，可以显示生成图片。
6. 图片接口失败时，页面不会崩溃，可以显示本地示意图或错误提示。
7. 多模态功能即使未接入真实大模型，也不影响基础阅读、线索收集和聊天助手流程。

### 实际结果

点击正文中的人物对话后，可以显示语音播放按钮。
点击播放后，系统可以调用语音生成接口，并播放生成的语音。
聊天助手可以成功调用真实大模型接口进行回答。
点击“生成场景图”后，系统可以调用图片生成接口，并在页面中显示生成图片。
页面未出现白屏、卡死或崩溃。

### 测试结论

通过。

## 前端手动测试总结

### 测试范围

本次前端手动测试覆盖以下核心功能：

1. 登录注册流程。
2. 章节阅读流程。
3. 线索收集流程。
4. 聊天助手流程。
5. 语音、图片、多模态功能流程。

### 测试说明

前端手动测试主要关注用户在浏览器中的实际操作体验，检查页面跳转、按钮点击、数据展示、错误提示和状态保存是否符合预期。

### 总体结论

本次前端手动测试主要覆盖登录注册、章节阅读、线索收集、聊天助手以及多模态功能入口。

测试结果表明，系统前端核心流程可以正常使用：用户可以完成登录和注册操作，登录后可以进入书架和阅读页面；章节内容能够正常展示，章节切换功能正常；阅读过程中可以收集线索，并在证物袋中查看已收集内容；聊天助手可以正常发送问题并显示回答。

在配置 MiniMax API Key 后，聊天助手已能够调用真实大模型接口进行回答，说明大模型聊天能力已成功接入。语音朗读和图片生成等多模态功能入口已在页面中保留，后续可在完整 API 环境下继续补充测试。

总体来看，前端主要功能流程运行正常，页面未出现白屏、卡死或核心功能不可用的问题。

本次前端手动测试结论为：核心功能通过，多模态扩展能力部分待进一步验证。
